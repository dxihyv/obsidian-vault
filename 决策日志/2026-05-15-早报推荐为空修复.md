## 优化（第二阶段 v4.3）：涨停板次日缩量过滤 + 板块分散度

### 问题
- Top推荐全是涨停/接近涨停股（+10%），追高风险极高
- 板块集中度高（化工/材料为主），一旦回调所有持仓同时受损
- `top = confirmed[:2]` 覆盖了策略C/B的选股逻辑（bug）

### 优化1：涨停板次日缩量过滤
逻辑：昨日涨停今日缩量（一字板特征），说明市场追高意愿不足，容易炸板回落
```python
# 约第1584行，confirmed排序后过滤
def get_kline_vols(code):
    k = get_sina_kline(code, days=5)
    if k and len(k) >= 2:
        return float(k[-1].get('volume', 0)), float(k[-2].get('volume', 0))
    return 0.0, 0.0

for stk in confirmed:
    today_vol, yesterday_vol = get_kline_vols(stk['symbol'])
    is_limit_up = today_pct >= 9.8
    is_thin = yesterday_vol > 0 and today_vol < yesterday_vol * 0.5
    stk['is_watch'] = is_limit_up and is_thin  # 缩量涨停=观察
```

### 优化2：板块分散度控制
同板块最多2只，防止集中风险
```python
def build_final_top(pool, confirmed_list):
    sector_count = {}; result = []
    for stk in pool:
        if stk.get('is_watch', False): continue
        sector = stk.get('sector', 'unknown')
        if sector_count.get(sector, 0) >= 2: continue
        sector_count[sector] = sector_count.get(sector, 0) + 1
        result.append(stk)
        if len(result) >= 2: break
    # 若不足2只，用非watch股补充
    if len(result) < 2:
        for stk in confirmed_list:
            if stk in result or stk.get('is_watch'): continue
            result.append(stk)
            if len(result) >= 2: break
    return result[:2]
```

### 优化3：修复bug `top = confirmed[:2]`
原来line 1629直接覆盖了策略C/B逻辑 → 改为使用 `final_top`

### 优化4：早报新增"观察区"
展示被过滤的缩量涨停股，标注"⚠️ [观察]"和原因，不推荐追但保留关注
