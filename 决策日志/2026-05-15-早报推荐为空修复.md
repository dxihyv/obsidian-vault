## 优化（第二阶段 v4.3）：涨停板次日缩量过滤 + 板块分散度

### 优化2：板块分散度控制
同板块最多2只，防止集中风险
```python
def build_final_top(pool, confirmed_list):
    sector_count = {}; result = []
    for stk in pool:
        if stk.get('is_watch', False): continue
        sector = stk.get('sector', 'unknown')
        if sector_count.get(sector, 0) >= 2: continue
        sector_count[sector] = sector_count.get(sector, 0) + 1
        result.append(stk)
        if len(result) >= 2: break
    # 若不足2只，用非watch股补充
    if len(result) < 2:
        for stk in confirmed_list:
            if stk in result or stk.get('is_watch'): continue
            result.append(stk)
            if len(result) >= 2: break
    return result[:2]
```

## 优化（第三阶段 v4.4）：大盘环境过滤

### 问题
系统对大盘整体涨跌没有判断，熊市里再好的技术信号都会被大盘拖垮。

### 优化：大盘环境分层处理
根据上证/深证/创业板/科创50的平均涨跌幅，自动调整仓位和推荐数量：

| 大盘状态 | 4指平均涨跌幅 | 仓位上限 | 新仓数量 | 操作建议 |
|---------|--------------|---------|---------|---------|
| 🟢强势 | > +1% | ≤10% | ≤2只 | 积极操作，顺势而为 |
| 🟡中性 | -1% ~ +1% | ≤5% | ≤2只 | 谨慎操作，优选强势股 |
| 🔴弱势 | -1% ~ -2% | ≤3% | ≤1只 | 轻仓防守，观望为上 |
| 🆘极弱 | < -2% | 0% | 0只 | 暂停新仓，专注持仓 |

## 优化（第四阶段 v4.4）：总仓位上限 + 修复新股未写入Bug

### 问题1：总仓位无上限
虚拟持仓可以无限增长，熊市时持仓越多亏损越大。

### 问题2：早报新股从未写入 pending_positions.csv
`write_top2_to_pending()` 函数定义了但从未被调用，导致早报推荐的新股没有进入虚拟持仓追踪。

### 优化：总仓位上限5只

## 优化（第五阶段）：满仓Rotation评估

### 问题
早报Top2在满仓（5只）时直接跳过，新股完全无法进入追踪。

### 解法：满仓时先做Rotation评估

当 `current_holdings >= max_total` 时，不直接跳过，而是调用 `_evaluate_rotation_for_stock(stk)` 读取 `sim_portfolio.json` 评估最弱持仓的信念分：

| 条件 | 结果 | pending状态 |
|------|------|------------|
| 信念<70 **且** 新股评分-信念≥5 | Rotation成功 | 持仓中 |
| 信念70-79 **且** 差距≥15分 | Rotation成功 | 持仓中 |
| 以上都不满足 | 无可替换空间 | 待仓位 |

### Rotation评估逻辑（`_evaluate_rotation_for_stock`）
1. 读取 `sim_portfolio.json`（模拟盘真实持仓）
2. 获取每只持仓的实时价格和信念分
3. 按信念分升序，最弱的排在前面
4. 检查最弱持仓是否满足替换条件
5. 返回替换说明（成功）或 `None`（不替换）

### 替换条件（来自sim_trading.py Rotation 4）
- 条件1：持仓信念分<70 **且** 新股评分比信念高≥5分
- 条件2：持仓信念分<80 **且** 差距≥15分

### 验证
- 满仓4只 + Rotation评估通过 → 2只新股写入 ✅
- Rotation失败 → 标记"待仓位"，继续追踪不买入 ✅
