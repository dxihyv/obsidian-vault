## 止损体系（2026-05-14 新增）

### 规则对比

|------|------------------|-----------|
| 固定止损 | -4.0%（ref×0.96） | -3.0%（ref×0.97） |

## 根因（3层叠加）

### 第一层：pct >= 2% 硬截断
`analyze_stock()` 对所有非 pending 股票要求今日涨幅 `>= 2%`，但 q9 量比扫描筛出的 20 只股票中有 12 只涨幅 < 2%（量比高 = 还没怎么涨 = 还没启动）。

导致这 12 只 q9 票直接 `return None`，根本没有进入评分流程。

### 第二层：RSI 阈值过严
原阈值 `RSI <= 70`，q9 筛出的票量比高 = 已涨一截，RSI 全在 81-89（超买区），全部在 70 阈值被卡死。

### 第三层：q9 综合评分 vs prescore 是两套独立系统
- q9 量比扫描有独立的综合评分（量比、换手率、PE 等）
- `load_candidates()` 用的是 `prescore` 预打分公式，两者完全不同
- 导致 q9 的高分票（score=80+）被 prescore 排序压制，q9 的量比优势完全没有传递到早报推荐

---

## 修复方案（三刀）

### 刀1：移除 pct >= 2% 硬截断
**位置：** `analyze_stock()` 基础过滤区

```python
# 修复前
if not is_pending and (today_pct < 2.0 or today_pct > 20.5):
    return None

# 修复后
if today_pct > 20.5:
    return None
```

**理由：** 质量把关交给 RSI/MACD/均线等综合指标，不依赖涨幅单一阈值。q9 量比扫描本身就是找"还没怎么涨但量比异常"的票，不应该用涨幅二次过滤。

---

### 刀2：RSI 阈值 70 → 80
**位置：** `analyze_stock()` 综合推荐区

```python
# 修复前
recommend = score >= 50 and rsi14 <= 70 and macd_strong

# 修复后
recommend = score >= 50 and rsi14 <= 80 and macd_strong
```

**理由：** RSI 80 才是真正的超买警戒线（传统技术分析标准），70 是启动/稳健分界线。从 70 放宽到 80 让更多高量比票进入推荐池。

---

### 刀3：prescore 加入 RSI 健康度因子
**位置：** `prescore_candidate()` 函数

```python
# 新增 RSI 健康度因子
rsi14 = c.get('rsi14', 70)
if rsi14 > 85:
    rsi_factor = 0.6   # 严重超买
elif rsi14 > 75:
    rsi_factor = 0.8   # 超买
elif rsi14 > 70:
    rsi_factor = 0.9   # 略超买
else:
    rsi_factor = 1.0   # 健康

# 汇总公式增加 rsi_factor
return base * mkt_factor * rsi_factor
```

**理由：** 昨日涨停票 pct≈10%，prescore 的 pct_score 被推高到 8 分（满分），而 q9 票 pct≈1%，pct_score 约 4.5 分。q9 的量比优势（volume_ratio 70-90）只能补回 2.5 分，总分仍被昨日涨停票压制。RSI 因子让高量比高 RSI 的 q9 票在 prescore 层面不会霸榜。

## 教训总结

1. **量比扫描和技术审核是两个独立系统**：q9 量比扫描的输出（综合评分）没有直接传递给早报 prescore，导致量比优势丢失
2. **pct 涨幅和 RSI 是矛盾指标**：量比高的票还没怎么涨（pct 低），但容易 RSI 高。原来的 pct>=2% 硬截断直接杀死了 q9 票进入分析流程的可能性
3. **数据时效性差异**：早报 08:50 和模拟盘 09:35 运行时，q9 候选股的 pct 数据不同（盘中 vs 盘前），但 score 评分在 today_candidates.csv 中是固定的，不受盘中 pct 变化影响

## 方案D（最终采用）：RSI空间系数加权（2026-05-14）

**核心思路：** score 衡量"综合质量"，RSI 衡量"还剩多少空间"。两者相乘，优先推荐"质量高且还有空间的票"。

**RSI空间系数：**

| RSI 区间 | 系数 | 含义 |
|---------|------|------|
| 45-55（启动区）| ×1.30 | 刚发动，上涨空间最大 |
| 55-65（稳健区）| ×1.15 | 趋势健康 |
| 65-75（偏热）| ×1.00 | 不加分不扣分 |
| >75（超买）| ×0.80 | 降权 |
| <45（超卖）| ×1.15 | 可能反弹，轻微加分 |

**通过条件：** `adjusted_score >= 50 AND RSI <= 80 AND MACD红柱`

## 教训总结

1. **score 和"上涨概率"不是同一件事**：score 衡量综合质量（信号多/资金强），RSI 衡量剩余空间。两者相乘才能选出"质量高且还有空间的票"
2. **q9 量比扫描的核心价值**：找的是"量比异常但还没怎么涨"的票，这类票 RSI 往往在 60-75，RSI 空间系数加权后仍低于 50 以下的启动区票
3. **耀皮玻璃/华西股份替代潍柴重机/浙江世宝的原因**：潍柴重机 RSI=53.5（比启动区上限55只差1.5点），浙江世宝原始score=60（MACD信号弱），两者经 RSI 系数调整后均低于耀皮/华西

## 策略C（当前采用）：量比异常优先，精准定位"主力吸筹但还没涨"的票

**与RSI系数加权的关系：** RSI空间系数是调整分数的乘数，策略C是筛选候选池的两层过滤器，两者配合使用。

**三层选股架构：**
1. **prescore**（预筛）：从160只候选中筛选出较优质的部分
2. **analyze_stock**（技术审核 + RSI空间加权）：RSI<80且score>=50的票通过审核，adj_score=score×RSI系数
3. **策略C筛选**（最终过滤）：量比>40 + RSI<70 + pct<5% + MACD红柱，确保"主力在买但还没涨"

**策略C筛选条件：**
```
量比 > 40   → 主力异常关注，是平时的40倍以上资金量
RSI < 70    → 还有空间，不是已经透支
pct < 5%   → 价格还没涨，不是追高
MACD > 0   → 趋势向上，不是反弹出货
```

**策略B补充条件（策略C不足2只时）：**
```
RSI < 55   → 启动区，还有明显空间
pct >= 5%  → 已有明显涨幅但RSI仍健康（昨日涨停/强势股）
```

**硬性保护：** Top 2中至少1只必须来自策略C（量比异常=主力在场）
